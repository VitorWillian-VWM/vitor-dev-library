# 🤝 Contribuindo para a Open Computer Science Library

Obrigado pelo interesse em contribuir com a **Open Computer Science Library**.

Nosso objetivo é construir uma biblioteca Open Source com livros, apostilas e materiais de estudo de alta qualidade sobre Ciência da Computação, Programação, Engenharia de Software e Inteligência Artificial.

Toda contribuição é bem-vinda.

---

# 📚 Como você pode contribuir

Você pode colaborar de diversas maneiras:

- 📖 Criando novos livros;
- ✍️ Melhorando livros existentes;
- 🐞 Corrigindo erros de escrita;
- 📑 Corrigindo exemplos de código;
- 💡 Sugerindo novos conteúdos;
- 🎨 Melhorando imagens e diagramas;
- 🌍 Traduzindo materiais;
- 📄 Melhorando documentações;
- 🔍 Revisando conteúdos técnicos.

---

# 🚀 Antes de começar

Antes de criar uma contribuição, recomendamos que você:

- Leia o README principal;
- Consulte o Roadmap;
- Leia o Código de Conduta;
- Verifique se já existe uma Issue relacionada.

---

# 📂 Estrutura do Projeto

Cada livro deve possuir sua própria pasta.

Exemplo:

```
books/
│
├── inteligencia-artificial-5-anos/
│   ├── images/
│   ├── slides/
│   ├── source/
│   ├── README.md
│   └── livro.pdf
│
├── linguagem-de-programacao/
│   ├── images/
│   ├── slides/
│   ├── source/
│   ├── README.md
│   └── livro.pdf
```

---

# 📘 Criando um novo livro

Utilize o template localizado em:

```
templates/book-template.md
```

Sempre procure incluir:

- Introdução;
- Sumário;
- Capítulos organizados;
- Imagens quando necessário;
- Exemplos práticos;
- Referências;
- README do livro.

---

# 📝 Criando novos capítulos

Para manter um padrão entre todos os livros, utilize:

```
templates/chapter-template.md
```

Cada capítulo deve possuir:

- título claro;
- explicação objetiva;
- exemplos;
- imagens quando necessário;
- linguagem acessível.

---

# 💻 Código

Caso seu conteúdo utilize exemplos de programação:

- utilize código bem formatado;
- explique o funcionamento;
- mantenha exemplos simples;
- prefira boas práticas.

---

# 📖 Escrita

Procure utilizar uma linguagem:

- clara;
- objetiva;
- didática;
- organizada;
- acessível para iniciantes.

Evite textos excessivamente acadêmicos quando houver uma forma mais simples de explicar o assunto.

---

# 🖼️ Imagens

Sempre que possível:

- utilize imagens próprias;
- crie diagramas;
- utilize fluxogramas;
- cite a fonte quando necessário.

Evite utilizar imagens protegidas por direitos autorais sem autorização.

---

# 🐞 Encontrou algum erro?

Abra uma Issue utilizando o template:

```
.github/ISSUE_TEMPLATE/bug_report.md
```

Descreva:

- onde ocorreu o erro;
- como reproduzir;
- qual seria o comportamento esperado.

---

# 💡 Tem uma nova ideia?

Abra uma Issue utilizando:

```
.github/ISSUE_TEMPLATE/feature_request.md
```

Explique sua sugestão da forma mais detalhada possível.

---

# 📚 Quer adicionar um novo livro?

Utilize o template:

```
.github/ISSUE_TEMPLATE/new_book.md
```

Informe:

- título;
- tema;
- descrição;
- público-alvo;
- objetivos.

---

# 🔀 Pull Requests

Antes de enviar um Pull Request:

- atualize sua branch;
- revise seu conteúdo;
- verifique erros de ortografia;
- mantenha o padrão do projeto;
- utilize o template disponível.

---

# ✅ Checklist

Antes de enviar sua contribuição confirme que:

- [ ] O conteúdo foi revisado.
- [ ] Não há erros de português.
- [ ] Os exemplos foram testados.
- [ ] As imagens estão corretas.
- [ ] O README foi atualizado quando necessário.
- [ ] O conteúdo segue o padrão do projeto.
